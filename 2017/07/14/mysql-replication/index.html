MySQL Replication 主从复制

发表于 2017-07-14 | 分类于 linux进阶 ， MySQL | | 热度 ℃ |

MySQL Replication实现了Master节点可读可写，Slave节点只读
这样的逻辑结构，大大降低了主节点的压力。
因为MySQL的主要压力都是读操作，而非写操作。
通过增加Slave节点的数量，负载均衡读操作，大大提高用户体验，提升产品竞争力。
主节点(Master)： 可读可写

从节点(Slave)： read only 只读





双主模式： 每个节点都读对方的二进制日志，都使用二进制日志
